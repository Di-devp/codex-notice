# 用户操作手册

## 2026-06-11 使用 Notice 监控 Codex 状态

### 重新安装 Codex Hooks
1. 打开 Notice。
2. 进入 `Providers` 页面。
3. 点击 `Install Codex Hooks`。
4. 安装完成后，新开一个 Codex 会话，或重启 Codex 客户端后再测试。

重新安装的目的是把 `UserPromptSubmit` 写入 `~/.codex/config.toml`。该事件用于判断“用户发出新消息，Codex 开始执行本轮任务”。

### 红绿灯小组件含义
- 跑马灯：最近 30 分钟内存在正在运行的 Codex 会话。
- 黄灯：存在待审批命令或待用户确认状态。
- 红灯：存在失败会话，需要关注。
- 绿灯：没有失败、没有待审批、没有运行中的任务；如果已有完成事件，表示已完成，否则表示就绪。

### 小组件操作
- 直接按住红绿灯小组件拖动即可移动位置。
- 右键小组件可进行置顶或隐藏操作。
- 小组件显示异常时，先确认 Notice 本地服务是否正常：`curl -sS http://127.0.0.1:3746/health` 应返回 `ok`。

### 飞书通知策略
- 仅在需要用户审批时发送飞书通知。
- 仅在一个会话中的任务成功结束后发送飞书通知。
- 普通工具调用成功、工具调用过程中的中间状态不发送飞书通知。

## 2026-06-16 查看 Codex 剩余用量

### 小组件显示内容
- 小组件第一行仍显示 Codex 会话状态，例如“运行中”“等待确认”“已完成”“就绪”。
- 小组件第二行显示状态详情。
- 如果本机 Codex 会话日志中存在 `rate_limits` 数据，小组件会额外显示用量摘要，例如：`5h 100% · 7d 100%`。
  - `5h` 表示 5 小时窗口。
  - `7d` 表示 7 天窗口。
  - 百分比表示该窗口的剩余额度。

### 数据刷新说明
- Notice 读取的是本机 `~/.codex/sessions` 与 `~/.codex/archived_sessions` 中最近一次 Codex `rate_limits` 记录。
- 如果 Codex 长时间没有产生新会话或新响应，显示的用量可能短暂滞后。
- 小组件异常裁切时，先确认已经退出旧 Notice 实例，并打开新打包的 `Notice.app`。

### 打包产物
- macOS App：`src-tauri/target/release/bundle/macos/Notice.app`
- DMG 安装包：`src-tauri/target/release/bundle/dmg/Notice_1.0.0_aarch64.dmg`

## 2026-06-18 同步代码到所有 Git 平台

### 上传前检查
1. 查看状态：`git status --short`
2. 运行前端构建：`pnpm build`
3. 运行 Rust 测试：`cargo test --manifest-path src-tauri/Cargo.toml`

### 推送命令
当前项目关联三个远端：

- GitHub：`git push origin main`
- Gitee：`git push gitee main`
- GitCode：`git push gitcode main`

### Gitee SSH 认证
如果 Gitee 推送提示 `Permission denied (publickey)`：

1. 确认公钥指纹：`ssh-keygen -lf ~/.ssh/codex_notice_gitee.pub -E sha256`
2. 加载私钥：`ssh-add ~/.ssh/codex_notice_gitee`
3. 重新推送：`git push gitee main`

## 2026-06-18 配置 Notice 同步 Mochi 桌宠

### 配置 Mochi 地址
1. Mochi 第一次使用或需要改 WiFi 时，先连接它自己的热点：`ClaWD-Mochi`。
2. 打开 `http://192.168.4.1/wifi`，配置家庭 WiFi。
3. 页面显示 `Room WiFi: connected at <局域网 IP>` 后，把电脑切回同一个家庭 WiFi。
4. 打开 Notice 设置页，在 `Mochi 桌宠` 中填写局域网地址，例如：`http://192.168.0.102`。
5. 打开 `开启桌宠同步`，点击 `Test pet` 验证。

注意：`192.168.4.1` 是 Mochi 自己热点地址，只适合电脑连接 `ClaWD-Mochi` 时使用；Notice 日常应填写家庭 WiFi 下的局域网地址。

### 余量显示
- Notice 会随桌宠状态一起发送 Codex 余量：
  - `quota`：5h 窗口，例如 `5h 88%`
  - `quota2`：7d 窗口，例如 `7d 41%`
- Mochi 固件会在宠物状态屏底部显示这两行余量。
- 如果状态和余量没有变化，Notice 会跳过重复同步，减少屏幕闪烁。

### 拍照状态覆盖
1. 打开 Notice 设置页。
2. 找到 `拍照状态覆盖`。
3. 直接选择 `就绪 / 运行中 / 等待确认 / 失败 / 已完成`。
4. 拍照完成后选择 `关闭覆盖`，恢复真实 Codex 状态。

拍照覆盖只改变小组件显示，不会写入事件历史，也不会污染统计。

### Mochi 烧录与编译
- Arduino CLI 已安装：`arduino-cli 1.5.1`。
- ESP32 core：`esp32:esp32 3.3.10`。
- 推荐 FQBN：`esp32:esp32:nologo_esp32c3_super_mini`。
- 常见端口：`/dev/cu.usbmodem1301`。
- 普通烧录不会清除 Preferences 中保存的 WiFi；不要勾选“Erase Flash / 擦除全部 Flash”，除非确实要重置配置。
