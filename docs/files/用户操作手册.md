# 用户操作手册

## 2026-06-11 使用 Notice 监控 Codex 状态

### 重新安装 Codex Hooks
1. 打开 Notice。
2. 进入 `Providers` 页面。
3. 点击 `Install Codex Hooks`。
4. 安装完成后，新开一个 Codex 会话，或重启 Codex 客户端后再测试。

重新安装的目的是把 `UserPromptSubmit` 写入 `~/.codex/config.toml`。该事件用于判断“用户发出新消息，Codex 开始执行本轮任务”。

### 红绿灯小组件含义
- 跑马灯：最近 30 分钟内存在正在运行的 Codex 会话。
- 黄灯：存在待审批命令或待用户确认状态。
- 红灯：存在失败会话，需要关注。
- 绿灯：没有失败、没有待审批、没有运行中的任务；如果已有完成事件，表示已完成，否则表示就绪。

### 小组件操作
- 直接按住红绿灯小组件拖动即可移动位置。
- 右键小组件可进行置顶或隐藏操作。
- 小组件显示异常时，先确认 Notice 本地服务是否正常：`curl -sS http://127.0.0.1:3746/health` 应返回 `ok`。

### 飞书通知策略
- 仅在需要用户审批时发送飞书通知。
- 仅在一个会话中的任务成功结束后发送飞书通知。
- 普通工具调用成功、工具调用过程中的中间状态不发送飞书通知。
