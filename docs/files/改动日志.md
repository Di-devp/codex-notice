# 改动日志

## 2026-06-11 Notice Hook、红绿灯小组件与配置修复

### 改动内容
- `src-tauri/src/hooks/mod.rs`：Notice 管理的 Codex hooks 事件列表包含 `SessionStart`、`UserPromptSubmit`、`PermissionRequest`、`PreToolUse`、`PostToolUse`、`Stop`。`status()` 现在会检查管理块是否完整；如果旧配置缺少 `UserPromptSubmit`，返回“需要重新安装”，避免误判为已安装。
- `src-tauri/capabilities/default.json`：为 `main` 与 `traffic-widget` capability 增加 `core:window:allow-start-dragging`，修复小组件调用 `getCurrentWindow().startDragging()` 时被 Tauri 2 权限系统拒绝的问题。
- `src/App.vue`：红绿灯小组件使用 `getCurrentWindow().startDragging()` 进行窗口拖动；运行态使用 `traffic-running` 类让红、黄、绿三灯按跑马灯动画轮询。
- `src-tauri/src/storage/mod.rs`：红绿灯状态基于最近 30 分钟的会话事件计算。失败优先于审批，审批优先于运行，全部完成后才显示绿色。
- `src-tauri/src/server/mod.rs`：使用 `UserPromptSubmit` 记录 `TASK_START`，使用 `Stop` 记录本轮任务结束；普通工具成功事件不再作为用户关心的运行状态展示。

### 验证情况
- `pnpm build` 通过。
- `cargo test --manifest-path src-tauri/Cargo.toml` 通过，当前 22 个 Rust 测试全部通过。
- `pnpm tauri build --debug` 通过，产物位于 `src-tauri/target/debug/bundle/macos/Notice.app`。
- 重启 Notice 后，`curl -sS http://127.0.0.1:3746/health` 返回 `ok`。

### 迁移说明
- 当前用户机器的 `~/.codex/config.toml` 旧 Notice 管理块缺少 `UserPromptSubmit`。需要在 Notice 的 Providers 页面重新点击安装 Codex Hooks，随后新开 Codex 会话或重启 Codex 客户端测试红绿灯运行态。
