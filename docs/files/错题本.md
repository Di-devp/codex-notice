# 错题本

## 2026-06-11 Tauri 2 窗口拖动 API 缺少 capability 权限

**问题现象：**
红绿灯小组件无法直接拖动，前端已经调用 `getCurrentWindow().startDragging()`，但窗口不移动。

**原因分析：**
Tauri 2 的前端 API 受 capability 控制。`src-tauri/capabilities/default.json` 中只配置了 `core:default`，没有授权 `core:window:allow-start-dragging`。即使窗口 label 已包含 `traffic-widget`，缺少具体权限时 `startDragging()` 仍会被拒绝。

**正确解法：**
在 `src-tauri/capabilities/default.json` 中为 `main`、`traffic-widget` 所属 capability 增加：

```json
"core:window:allow-start-dragging"
```

修改后必须重新执行 `pnpm tauri build --debug` 并重启 `.app`，否则当前运行实例仍使用旧 capability。

## 2026-06-11 Hook 状态不能只看 Notice 管理块存在

**问题现象：**
Providers 页面显示 hooks 已安装，但红绿灯始终保持绿色，无法在用户开始新一轮对话时进入跑马灯。

**原因分析：**
旧的 `~/.codex/config.toml` Notice 管理块只包含 `SessionStart`、`PermissionRequest`、`PreToolUse`、`PostToolUse`、`Stop`，缺少 `UserPromptSubmit`。应用之前只检查 Notice 管理块起止标记是否存在，没有检查关键事件是否齐全，导致旧配置被误判为可用。

**正确解法：**
`src-tauri/src/hooks/mod.rs` 中的 `status()` 应检查管理块是否完整，尤其是 `UserPromptSubmit`。缺失时提示需要重新安装 hooks，而不是显示已安装。

## 2026-06-11 任务级状态不能用工具调用级事件判断

**问题现象：**
用户只关心“这一轮 Codex 是否开始执行、是否需要确认、是否失败、是否成功结束”，但旧逻辑把 `PreToolUse` / `PostToolUse` 当作运行状态依据，导致一次会话多次工具调用时事件噪声过大、状态不符合用户预期。

**原因分析：**
工具调用是任务执行过程中的内部细节，不等价于用户感知的一轮任务开始或结束。

**正确解法：**
使用 `UserPromptSubmit` 表示本轮任务开始，使用 `Stop` 表示本轮任务结束；普通成功的工具调用不作为 Events 的主要展示事件，也不触发飞书通知。

## 2026-06-16 小组件内部变宽后仍会被旧根容器裁切

**问题现象：**
为红绿灯小组件增加 Codex 用量显示后，截图中右侧被硬切，文字和背景都不完整。

**原因分析：**
只改了 `.traffic-widget` 和 Tauri `WebviewWindowBuilder` 的窗口尺寸，但全局 `src/styles.css` 中 `body.widget-body #app` 仍固定为旧尺寸 `164px x 66px`。Tauri WebView 根节点会先裁掉内部内容，导致右侧显示不完整。

**正确解法：**
同步更新所有尺寸来源：

- `src/styles.css` 的 `body.widget-body #app`。
- `src/App.vue` 的 `.traffic-widget` 宽高。
- `src-tauri/src/commands/mod.rs` 的 Tauri 窗口 `inner_size` / `min_inner_size` / `max_inner_size`。

另外，复用已存在的 `traffic-widget` 窗口时必须主动调用 `set_size`，否则旧实例可能继续使用历史窗口尺寸。

## 2026-06-16 DMG 打包可能需要脱离沙箱运行

**问题现象：**
`pnpm tauri:build` 在前端构建和 release 编译都成功后，失败在 `bundle_dmg.sh` 阶段。

**原因分析：**
Tauri macOS DMG 打包会调用 `hdiutil` 和相关系统能力；在当前受限沙箱内可能无法完成 DMG 制作。

**正确解法：**
先确认 `.app` 已生成；如果确实卡在 DMG 阶段，使用同一命令在非沙箱权限下重新执行 `pnpm tauri:build`。本次非沙箱重跑成功生成 `Notice_1.0.0_aarch64.dmg`。

## 2026-06-16 understand-anything 扫描不要纳入本地生成缓存

**问题现象：**
运行 understand-anything deterministic scan 时，文件数从旧记录的 52 激增到 188，复杂度被判定为 `large`，语言统计里出现大量 `.pcm`、`.swiftmodule` 等缓存格式。

**原因分析：**
`.understand-anything/.understandignore` 没有忽略 `.tmp/` 和 `.codegraph/`。当前项目里 `.tmp/clang-module-cache/` 包含大量构建过程缓存，扫描脚本会按文件扩展名把它们纳入项目代码统计。

**正确解法：**
在 `.understand-anything/.understandignore` 中加入：

```text
.codegraph/
.tmp/
```

然后重新运行 scan/import/batch/fingerprint 脚本。修正后本项目扫描结果为 88 个项目文件、过滤 100 个生成缓存文件。
